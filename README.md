# Site Clone — Laboratório de Estudo em Cybersegurança / Cybersecurity Learning Lab


## Português (PT-BR)

**Descrição curta**  
Repositório com um clone de site usado **apenas** para estudo em ambiente de laboratório/bootcamp. Destina-se ao aprendizado teórico e prático **autorizado** sobre análise de tráfego, inspeção de formulários e investigação forense.

**Resumo do fluxo utilizado (ALTO NÍVEL — NÃO ACIONÁVEL)**  
- Ambiente: máquina isolada/VM em rede de laboratório, com autorização explícita para testes.  
- Fluxo (conceito): foi utilizada uma ferramenta de simulação para demonstrar um cenário de engenharia social voltado à análise de tráfego e coleta de logs. Em termos gerais, o exercício consistiu em:  
  1. executar a ferramenta em ambiente local com privilégios administrativos;  
  2. selecionar um módulo de engenharia social / website attack (conceito);  
  3. escolher um método de captura de credenciais para fins de estudo (apenas em ambiente controlado);  
  4. fornecer parâmetros de laboratório (ex.: IP da máquina de teste e a URL alvo no laboratório);  
  5. observar/registrar acessos e credenciais em tempo real para análise posterior.  

**Aviso legal e ético**  
Este material é **exclusivamente educacional**. Não publique informações sensíveis sem redigir. Não reproduza essas técnicas contra sistemas sem permissão — phishing, clonagem de sites e captura de credenciais são ilegais e antiéticos. Sempre obtenha autorização por escrito e realize testes apenas em ambientes controlados.

## English (EN)

**Short description**  
Repository with a site clone used **only** for study in a lab/bootcamp environment. Intended for authorized learning on traffic analysis, form inspection and forensic investigation.

**High-level flow used (NON-ACTIONABLE)**  
- Environment: isolated VM / lab network with explicit permission to test.  
- Flow (conceptual): a simulation tool was used to demonstrate a social-engineering scenario focused on traffic analysis and log collection. In general terms, the exercise involved:  
  1. running the tool on a local, authorized machine with administrative privileges;  
  2. selecting a social-engineering / website attack module (conceptual);  
  3. selecting a credential-harvesting method for study purposes only (lab only);  
  4. supplying lab parameters (e.g., test machine IP and lab target URL);  
  5. monitoring/logging accesses and captured credentials in real time for later analysis.  

**Legal & ethical notice**  
This is strictly educational. Do not publish sensitive data without redaction. Do not perform these techniques against systems you do not have permission to test — phishing, site cloning and credential harvesting are illegal and unethical.
